# Project Conventions

---

## Directory Structure

```
project/
├── data/              # Raw and processed data (gitignored — never commit)
├── notebooks/         # Exploratory Quarto notebooks (POCs, not production)
├── scripts/           # Production analysis scripts (Python)
├── manuscript/        # LaTeX source for the paper
├── output/            # Script-generated artifacts committed to git
│   ├── figures/       # PDF figures produced by scripts
│   └── tables/        # LaTeX table fragments produced by scripts
└── _output/           # Quarto render output (gitignored — do not touch)
```

---

## Folder Conventions

### `data/`
- Raw data exports (CSV, DuckDB, etc.) live here.
- **Gitignored.** Never commit data files.

### `notebooks/`
- Quarto `.qmd` notebooks for quick, exploratory proof-of-concepts.
- Use these to test ideas, explore data, and prototype visualizations.
- **Do not put production analysis code here.** Once a notebook analysis is ready for the paper, port it to `scripts/`.
- **Naming convention:** `YYYY-MM-DD_purpose.qmd`
- Render with: `uv run quarto render notebooks/<name>.qmd`
- Rendered HTML output goes to `_output/notebooks/` (gitignored).

### `scripts/`
- Production Python scripts that feed into the final paper.
- Each script should be self-contained and reproducible: it reads from `data/`, performs analysis, and writes outputs to `output/figures/` or `output/tables/`.
- Run scripts with: `uv run python scripts/<name>.py`
- Utility modules live here and are imported by other scripts.

### `manuscript/`
- LaTeX source files for the paper.
- `main.tex` is the entry point; section files (`1-intro.tex`, etc.) are `\input{}`'d from it.
- `submission.qmd` is a Quarto wrapper for DOCX export when needed.
- **Do not manually create figures or tables here.** They are generated by scripts and imported.

### `output/`
- **Git-tracked.** Commit generated figures and tables so the manuscript can compile without re-running scripts.
- Contents are always generated by scripts — **do not edit these files by hand.**
- `output/figures/` — PDF figures (and PNG copies for DOCX/presentations)
- `output/tables/` — LaTeX `.tex` table fragments and CSV copies

---

## Scripts-to-Manuscript Pipeline

### Figures

Scripts save figures as **PDF** to `output/figures/`:

```python
fig.savefig("output/figures/fig-name.pdf", bbox_inches="tight")
```

In LaTeX, include them using just the filename (`\graphicspath` is set in `main.tex`):

```latex
\begin{figure}[H]
  \centering
  \includegraphics[width=\linewidth]{fig-name.pdf}
  \caption{...}
  \label{fig:name}
\end{figure}
```

### Tables

Scripts save tables as **LaTeX `.tex` fragments** to `output/tables/`:

```python
df.to_latex(
    "output/tables/tab-name.tex",
    index=False,
    booktabs=True,
    caption="...",
    label="tab:name",
    position="H",
)
```

In LaTeX, include them with `\input{}`:

```latex
\input{../output/tables/tab-name.tex}
```

### Naming Convention

| Type    | Prefix  | Example                         |
|---------|---------|---------------------------------|
| Figure  | `fig-`  | `fig-network-visualization.pdf` |
| Table   | `tab-`  | `tab-descriptive-stats.tex`     |

Names should be lowercase, hyphen-separated, and descriptive.

## Tooling

| Task                        | Command                                      |
|-----------------------------|----------------------------------------------|
| Run a Python script         | `uv run python scripts/<name>.py`            |
| Render a notebook           | `uv run quarto render notebooks/<name>.qmd`  |
| Render all notebooks        | `uv run quarto render`                       |
| Install/sync dependencies   | `uv sync`                                    |
| Compile manuscript          | `cd manuscript && latexmk -xelatex main.tex` |
| Export DOCX                 | `uv run quarto render manuscript/submission.qmd` |

---

## What NOT to Do

- **Don't put production analysis in `notebooks/`** — notebooks are for exploration only.
- **Don't manually edit files in `output/`** — they are generated by scripts.
- **Don't commit files in `data/`** — data is gitignored for size and privacy reasons.
- **Don't save figures or tables anywhere other than `output/`** — the manuscript assumes these paths.
- **Don't save figures as PNG for the manuscript** — use PDF for publication-quality vector output.
